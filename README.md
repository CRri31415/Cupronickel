# Cupronickel


본 문서는 제작자 본인이 직접 작성한 문서로, 인공지능이 개입하지 않았다.
본 문서는 기획서이며 설명서인 동시에 인공지능에게 입력할 프롬프트이다.

## 쿠프로니켈에 대해
쿠프로니켈은 Obsidian, Anki, Geany에서 영향을 받은 통합 소프트웨어이다.
그 이름은 저온 가열코일에 사용되기도 하는 합금인 백동에서 유래했다.

쿠프로니켈의 핵심 설계철학은 다음과 같다.
- 사용자의 생각을 장벽 없이 표현하고 정리한다.
- 사용자가 본인만의 생각에 몰입할 수 있도록 한다.
- 다양한 최적화 옵션으로 초저사양 선택지를 제공한다.

위 설계 철학에 의하여 다음 핵심 특징을 가진다.
- UI는 단순하면서 범용적으로 설계된다.
- 인공지능의 보조는 의도적으로 배제된다.
- 그래픽 프레임워크는 Tauri를 사용한다.
- Svelte + Vite를 사용한다.
- 기능들을 모듈화하여 사용자가 원하는 기능만 사용할 수 있도록 한다.
- 사용되지 않는 기능에 의한 메모리 점유나 성능 하락은 발생해선 안 된다.

쿠프로니켈은 설치 프로그램으로 배포되는 소프트웨어이다. Program Files 폴더에 Cupronickel 디렉토리를 생성하도록 한다.
권한 문제가 있을 것을 대비하여 앱 데이터는 앱 데이터 폴더로 넣도록 한다.
설치하지 않은 모듈들은 언제든 설치할 수 있도록 Cupronickel 폴더 안에 압축하여 저장해둔다.

쿠프로니켈은 반드시 인터넷 연결이 없는 상태에서도 사용 가능해야한다.
인터넷 하이퍼링크 실행같이 상식적으로 인터넷 연결이 필요한 경우를 제외하여,
인터넷 연결 부재에 의한 비합리한 문제 발생은 일어나선 안 된다.

쿠프로니켈은 바이브 코딩에 의해 생성될 프로젝트이다.
따라서 보안 또는 시스템에 관련된 안정성을 보장할 수 없다.
이를 해결하기 위해 쿠프로니켈은 GPL v2로 배포될 예정이다.
GPL v3을 사용하지 않는 것은 제작자 본인의 의사이다.
소스코드 접근과 함께, 쿠프로니켈 정식 배포에서 이 문서에 대한 접근은 보장되어야 한다.

### 기타 사항
개발 편의성과 성능 향상을 위해 해상도는 1920x1080 또는 1600x900만 지원한다.
윈도우에서 사용하는 PC 소프트웨어이다.
맥 지원은 고려하지 않는다. 우분투 지원은 고려할 수 있다.

인공지능은 쿠프로니켈을 생성할 때 이 문서에 담기지 않은 기능을 자율적으로 추가하는 것을 불가피하지 않다면 자제해야 한다.
그러나 보안상의 취약점이 있다면 자율적으로 보완하여 명시해야 한다.
인공지능은 쿠프로니켈을 생성할 때 이 문서에 담기지 않은 기능을 추가했다면 그것들을 확실히 명시해야한다.

인공지능은 인간의 유지보수가 쉽도록 적당한 주석을 달아야 한다.
인공지능은 코드의 흐름 전체를 요약할 수준의 주석을 달 필요는 없다.
그러나 어떤 코드가 어떤 방식으로 작동하는지의 흐름을 정리한 문서를 정리하여 생성해야 한다.
인공지능은 인간의 유지보수가 쉽도록 코드의 흐름을 가능한 한 복잡하지 않게 해야한다.

인공지능은 쿠프로니켈 설치프로그램을 빌드하여 제공한다.
그리고 의존 환경을 최소화 하는 방향으로 개발하라.
## UI 레이아웃
크롬의 디자인과 유사하게, 탭을 사용하며 분할화면 기능이 있다.
탭들은 창 최상단에 크롬과 유사하게 표시된다.
창 좌측에는 메인화면 버튼, 설정 버튼, 각 기능들을 여는 버튼들이 있는 세로줄이 있다.
### 디자인
디자인은 전반적으로 최대한의 미니멀리즘을 지향한다.
그리고 색은 카세트 퓨처리즘 스타일의 Earth Tone을 사용한다. 채도는 낮아야한다.
색은 테마 설정을 통해 사용자가 직접 바꿀 수 있어야 한다.

탭을 열고 닫거나 버튼을 누르는 등 상호작용에서 애니메이션 기능을 제공한다.
그러나 이는 최적화를 위해 끌 수 있어야 하며, 끌 경우 아예 없었던 것 처럼 확실한 최적화가 보장되어야 한다.
## 기능
메인화면 그 자체를 제외한 거의 모든 기능들은 모듈들로 제공되며, 사용자가 설치하여 사용할 지 직접 선택할 수 있어야 한다.
메인화면에 종속된 기본 기능을 제외하고 모듈 기능은 아래와 같다.
- 노트
- 암기
- 메모
- 달력
- 개발용 텍스트 에디터
- 그래프 다이어그램
- 시트

몇몇 모듈들은 Cupronickel 폴더 아래에 개별 폴더를 가진다.
노트는 note, 암기는 card, 메모는 memo, 달력은 calendar, 텍스트 에디터는 code, 그래프 다이어그램은 graph, 시트는 sheet 폴더를 가진다.
또한, 세팅이나 리소스, 메인화면 위젯 설정 등 모듈과 관계 없는 쿠프로니켈 자체의 구동에 필요한 파일들은 Cupronickel 하위의 app 폴더에 저장하도록 한다.
설치하지 않은 모듈들의 압축파일은 module 폴더에 저장하고, module 폴더에 있던 모듈을 설치했을 경우 module 폴더에선 삭제되도록 한다.

암기와 달력 탭은 한번에 두 개 이상 켜질 수 없다.
노트, 메모, 텍스트 에디터, 그래프 다이어그램, 시트 등은 서로 다른 두 개 이상의 탭이 하나의 파일을 편집하거나 열람하게 하지 않도록 해야한다.

모듈들은 이후 업데이트, 수정 ,추가 등에 대응할 수 있도록 일정 규격을 따라야한다.
이를 위해 쿠프로니켈은 모듈에서 사용할 수 있는 자체 프레임을 만들어 제공할 수 있다.

열려있는 탭들은 활성 상태와 비활성 상태가 있다.
사용하고 있거나 사용한지 일정 시간이 지나지 않은 탭은 활성 상태로, 메모리에 올라간다.
사용한지 일정 시간이 지나거나 쿠프로니켈이 꺼져있을 때의 모든 탭은 비활성 상태가 되어 하드디스크에 올라간다.
따라서 쿠프로니켈을 다시 켜면 이전에 보던 탭 상태 그대로 켜진다.

폰트나 색상 구성은 사용자가 직접 설정하고 그 테마를 저장하여 한번에 변경할 수 있다.

### 메인화면
메인화면은 메모, 위젯을 배치할 수 있는 쿠프로니켈 기본 화면이다.
메인화면의 배경사진은 사용자가 직접 변경할 수 있다.
### 위젯
위젯은 메인화면에 배치하여 사용한다.
#### 시계
시계는 아날로그와 디지털을 선택 가능하며, 다양한 그래픽 옵션을 제공한다.
12/24 옵션, 몇가지 폰트들을 제공해야 한다.
#### 디데이
디데이 위젯은 일정을 선택하여 디데이를 표시하는 위젯이다.
#### 달력
달력 위젯은 작은 달력을 표시한다. 히트맵 기능을 지원한다.
#### 인용구
인용구 위젯은 사용자가 직접 문장과 저자를 입력하여 만든 인용구 목록에서 랜덤으로 하나를 골라 보여주는 위젯이다.
### 노트
노트 모듈은 마크다운 형식의 노트를 생성하고 폴더별로 정리할 수 있는 기능이다.

노트는 각각 전용 폴더를 가진다.
예를 들어 “과학” 폴더 안에 “전자”, “세포” 노트가 있다면 전자.md의 디렉토리 구조는
과학\전자\전자.md 가 된다. 즉, 마크다운 파일 하나당 전용 폴더를 하나씩 가진다.

지원하는 문법은 아래와 같다.
- \# 문단
- \- 또는 \* 리스트
- \** \** 또는 \__ \__ 볼드체
- \* \* 또는 \_ \_ 기울임체
- \> 인용블록
- \`\`\` 고정폭 폰트 블록(문법 하이라이팅 없음)
- \[^] 각주
- !\[] 이미지
- \[\[]] 문서간 하이퍼링크

가로줄은 잘 사용되지 않기에 지원하지 않는다.
인터넷 하이퍼링크는 대괄호 입력을 불편하게 하기에 지원하지 않는다.
URL을 그대로 쓰면 인식하도록 하여 대체할 수 있다.
표는 구현이 복잡하고 편의성이 좋지 않아 지원하지 않는다.
표는 시트 기능에서 제작하여 이미지 형태로 넣도록 한다.

이미지는 사용자가 직접 이미지의 경로를 설정해야 하는것이 아니라 이미지 삽입으로 한다.
이미지 삽입은 사용자가 컴퓨터나 클립보드에서 이미지를 넣으면, 그 이미지가 노트가 있는 폴더에 저장되어, 저장된 이미지에 접근하는 이미지 링크를 만들어 주는 방식으로 작동한다.
따라서 사용자가 노트를 삭제할 때는 삭제할 노트를 제외한 다른 노트에서 사용되지 않는 이미지들이 함께 삭제되어야 한다.
상대경로로 연결하며 이미지와 노트가 같은 폴더에 있기에 이미지 파일 이름만 입력해도 된다.

문서간 하이퍼링크는 상대경로로 다른 .md 문서에 접근하게 한다.
그러나 note 폴더 밖에 있는 파일에 접근할 수 있게 해선 안 된다.

### 텍스트 범용 확장
텍스트 범용 확장 모듈은 다른 모듈들에 대해 다양한 확장 기능들을 제공한다.

제공하는 기능은 아래와 같다.
- \$ 기호를 사용한 TeX 문법
- \`\`\`smiles 코드블록을 사용한 SMILES 렌더링

텍스트 범용 확장 모듈은 노트, 암기, 메모, 그래프 다이어그램 모듈에 영향을 준다.
그리고 추후 모듈 추가에 대응하기 위하여 이는 텍스트 범용 확장 모듈이 설치되어있다면 다른 모듈에서 호출하여 사용할 수 있게 되는 방식으로 작동해야 한다.

### 암기
암기 모듈은 Anki와 같은 방식으로 암기 카드를 만들고 리뷰할 수 있게 한다.
암기 카드는 card 폴더에 저장된다.
모든 카드는 Anki의 Basic 유형 카드와 같이 앞면과 뒷면을 가진다.
카드를 리뷰할 때는 카드의 앞면이 나타나고, 스페이스 바 또는 엔터를 누르면 카드의 뒷면이 나타남과 동시에 난이도 피드백을 할 수 있다.
난이도 피드백은 아래에 네 개의 버튼으로 나타나고, 1,2,3,4 숫자키로도 할 수 있다.
숫자가 클 수록 쉽다는 것을 의미하고, 쉽다고 피드백된 카드는 알고리즘에 의해 더 긴 시간 뒤에 리뷰된다.
암기 알고리즘은 FSRS를 사용한다.
카드의 앞면과 뒷면이라는 단어 선택은 단지 상징적인 명칭이다. 실제 앞면과 뒷면을 의미하는 것은 아니며, 그것이 디자인에 반영되어야 하는 것도 아니다.

### 메모
메모 모듈은 포스트잇을 만들어 붙이거나 보관하는 기능이다.
포스트잇은 280자 내로 텍스트를 쓸 수 있다.
이를 메인 화면에 최대 16개까지 붙이거나 메모서랍이라는 보관 기능에 보관할 수 있다.

### 달력
달력 모듈은 일정을 관리하고 히트맵을 만드는 기능이다.
일정은 중요도에 따라 4단계로 구분하여 설정할 수 있다.
또한 일정은 특정 태그로 분류할 수 있고, 일정들은 태그별 필터, 날짜순 정렬 등으로 검색할 수 있다.
일정은 하루 길이로만 할 수 있는것이 아니라 여러날에 걸쳐 설정할 수 있다.
또한 어떤 일정의 후속 일정을 설정할 수도 있다.
일정에 대한 간략한 설명을 텍스트로 적을 수 있다. 여기에는 텍스트 범용 확장 모듈이 적용된다. 그러나 일정의 제목에는 텍스트 범용 확장 모듈이 적용되지 않는다.

### 개발용 텍스트 에디터
텍스트 에디터 모듈은 간단한 코딩을 위해 사용할 수 있는 텍스트 에디터이다.
프로젝트별로 구분할 수 있고, 한 프로젝트는 한 폴더에 저장된다.
제공되는 언어는 C99, Python3, Haskell이다.
그러나 쿠프로니켈에서 실행 환경의 설치를 보장하는 것은 아니다.
사용자가 직접 GCC, Python, GHCup 등의 실행 환경을 설치해야 한다.
그 이후 쿠프로니켈은 `ghc --make "%f"` 나 `.\"%e"`등의 명령을 실행하여 설치된 실행환경이 프로그램을 빌드하고 실행하게 한다.

텍스트 에디터에서는 vi 키맵을 일부 사용할 수 있다.
그러나 사용자가 vi 키맵을 알고 있다는 것을 확인하기 위하여, 종료 명령을 쓸 수 있는지 확인하는 등의 절차를 거쳐야 한다.
모든 vi 명령어가 사용 가능한 것은 아니다. 몇몇 주요 명령어만 구현되었다.

줄 번호는 기본적으로 하이브리드 상대줄번호로 표기되며, 사용자는 설정에서 이를 절대줄번호로 변경할 수 있다.
자동완성이나 실시간 린트 등은 편의성과 성능을 위하여 굳이 구현하지는 않는다.

노트 기능에서 \`\`\` 코드블록 문법 하이라이팅을 사용하기 위해서는 텍스트 에디터 확장을 설치해야 한다.

텍스트 에디터의 색상 테마와 폰트도 사용자가 직접 설정할 수 있다.

### 그래프 다이어그램
그래프 다이어그램 모듈은 Obsidian의 그래프 뷰와 유사한 방식으로 다이어그램을 그릴 수 있다.
그래프 다이어그램은 그래프 보드라고 불리는 파일을 사용한다.
우선 도형 모양과 색을 지정할 수 있고 텍스트를 써넣을 수 있는 노드가 있다.
이 노드는 Obsidian의 그래프 뷰처럼 기본적으로 서로 밀어내며, 중심을 향하는 힘도 받는다.
노드들은 간선을 통해 서로 연결될 수 있고, 굵기를 지정할 수 있다.
간선으로 연결된 노드는 서로 끌어당겨지는 힘이 추가로 작용하여, 노드간 작용하는 기본적인 척력과 평형을 이루며 가까운 거리를 유지한다.

원할 경우 노드들을 격자상에 고정시킬 수 있고, 캡처하여 이미지로 내보낼 수도 있다.

그래프 다이어그램은 이러한 방식으로 노드들을 연결한 그래프를 만들고, 이를 통해 간단하고 직관적으로 다이어그램을 만들 수 있게 된다.

### 시트
시트 모듈은 간단하고 작은 표를 만드는데만 특화된 모듈이다. csv 파일이나 xlsx 파일 등을 불러오는 것은 지원되지 않는다.
기본적으로 적용되는 서식은 논문에서 주로 사용되는 양식, 즉 세로줄은 없고 맨 위와 맨 아래에만 굵은 줄 또는 두 줄 가로줄이 있고 타이틀 등과 데이터를 구분하는 비교적 얇은 가로줄이 있으며 텍스트는 Times New Roman 가운데 정렬로 쓰고 그래프는 무채색으로 색칠할 수 있는 양식을 적용한다.
그러나 그래프의 무채색을 색칠을 제외한 다른 양식은 자유롭게 변경할 수 있다.
표는 이미지 파일로 내보내진다.

## 설치법

### 🟢 컴퓨터를 잘 몰라도 되는 설치 방법

설치 파일(.exe)을 얻는 방법은 두 가지입니다. **둘 중 하나만** 하면 됩니다.

#### 방법 A — 내 컴퓨터에서 한 번에 만들기 (더블클릭)

1. 이 폴더 안의 **`BUILD-INSTALLER.bat`** 파일을 **더블클릭**합니다.
2. "이 앱이 변경하도록 허용하시겠어요?" 같은 관리자 창이 뜨면 **예**를 누릅니다.
3. 필요한 프로그램(Node, Rust, C++ 빌드 도구 등)이 **자동으로 설치**된 뒤,
   Cupronickel 설치 파일이 만들어지고 **그 폴더가 자동으로 열립니다.**
4. 폴더 안의 `Cupronickel_0.2.0-alpha_x64-setup.exe` 를 더블클릭하면 설치가 진행됩니다.

> 처음 한 번은 프로그램들을 받느라 몇 분 걸릴 수 있습니다.
> "방금 설치되어 아직 인식되지 않습니다"라는 안내가 나오면, 창을 닫고 `BUILD-INSTALLER.bat`를 **한 번 더** 더블클릭하세요.

#### 방법 B — GitHub가 대신 만들어 주기 (내 컴퓨터에 아무것도 설치 안 함)

1. 이 프로젝트를 GitHub 저장소에 올립니다.
2. 저장소 **Actions** 탭 → **build-installer** → **Run workflow** 를 누릅니다.
3. 빌드가 끝나면 결과의 **Artifacts**에서 `Cupronickel-Windows-Installer`를 내려받습니다.
   압축을 풀면 설치 파일(.exe)이 들어 있습니다.

#### 설치 파일을 실행하면

표준 설치 마법사(한국어)가 뜹니다. **다음 → 다음 → 설치**만 누르면
**`C:\Program Files\Cupronickel`** 에 설치되고 시작 메뉴에 등록됩니다.
실행에 필요한 WebView2는 Windows 11과 대부분의 Windows 10에 이미 들어 있습니다(없으면 위 빌드 과정에서 설치됨).

### 개발자용: 직접 실행/빌드

사전 요구사항(방법 A의 스크립트가 자동 설치하는 것과 동일): **Node.js 18+**, **Rust(rustup)**,
Windows는 **MSVC C++ Build Tools** + **WebView2**. Ubuntu는 `libwebkit2gtk-4.0-dev` 등.

```bash
npm install
npm run tauri dev      # 데스크톱 창으로 실행(핫리로드)
npm run dev            # 백엔드 없이 UI만 빠르게 보기(http://localhost:1420)
npm run tauri build    # 설치 프로그램 직접 빌드
```

빌드 산출물: Windows는 `src-tauri/target/release/bundle/nsis/*.exe`,
Ubuntu는 `bundle/deb/` 또는 `bundle/appimage/`.
